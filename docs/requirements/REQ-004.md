## 背景

实验室成员使用多种 AI 服务（ChatGPT / Codex、Claude / Claude Code），这些服务通常存在使用额度限制和冷却窗口机制（如 5 小时滑动窗口）。用户需要一个集中的面板来查看各 AI 服务的当前使用量和剩余额度，以便合理安排使用时间，避免在关键时刻触及限额。

## 用户故事

- 作为实验室成员，我希望在面板上看到我的 ChatGPT / Codex 使用额度消耗情况，以便知道还能用多少。
- 作为实验室成员，我希望在面板上看到我的 Claude / Claude Code 使用额度消耗情况，以便合理安排高强度使用的时间段。
- 作为实验室成员，我希望了解 5 小时滑动窗口的剩余时间和额度恢复预估，以便安排工作节奏。

## 功能需求

### ChatGPT / Codex 额度监控

- [ ] 展示 ChatGPT 当前使用量与限额
- [ ] 展示 Codex 当前使用量与限额
- [ ] 展示 5 小时滑动窗口的当前状态
- [ ] 展示额度恢复的预估时间

### Claude / Claude Code 额度监控

- [ ] 展示 Claude 当前使用量与限额
- [ ] 展示 Claude Code 当前使用量与限额
- [ ] 展示 5 小时滑动窗口的当前状态
- [ ] 展示额度恢复的预估时间

### 通用展示

- [ ] 使用进度条或仪表盘直观展示额度消耗比例
- [ ] 额度接近上限时提供视觉警告（如变红）
- [ ] 数据应定时刷新

## 非功能需求

- **性能**: 遵循 REQ-001 的轻量级约束
- **兼容性**: 遵循 REQ-001 的多终端适配要求
- **安全性**: AI 服务的 API 凭证/会话信息需安全存储；额度数据可能涉及用户隐私
- **准确性**: 额度数据应尽可能接近实时，误差在可接受范围内

## 验收标准

- [ ] 面板正确展示至少一个 AI 服务的当前使用量
- [ ] 5 小时滑动窗口的剩余时间展示正确
- [ ] 额度接近上限时有视觉警告
- [ ] 数据刷新后展示最新的额度信息

## 开放问题

- **数据源是最大的不确定性**：ChatGPT/Claude 是否提供官方 API 来查询使用额度？
  - ChatGPT/Codex：OpenAI 目前是否开放了使用量查询 API？如果没有，是否需要通过浏览器扩展/爬虫方式获取？
  - Claude/Claude Code：Anthropic 是否提供使用量查询 API？还是需要通过其他方式追踪？
  - 如果没有官方 API，是否接受用户手动输入/更新额度信息？
- "5 小时窗口" 的具体机制：是滑动窗口（最近 5 小时的累计使用量）还是固定窗口（每 5 小时重置）？不同服务的窗口机制是否相同？
- 是否需要监控多个用户/账号的额度？还是只监控一个共享账号？
  - 如果多用户：每个用户需要配置自己的凭证？还是有一个管理员统一配置？
- 额度的单位是什么？Token 数？消息条数？请求次数？不同服务可能不同。
- 是否需要历史使用趋势？（如过去 7 天的使用量折线图）
- ChatGPT Plus/Team/Enterprise 不同订阅级别的限额不同，需要支持哪些？
- Claude Pro / Team / Max 不同订阅级别的限额不同，需要支持哪些？
- 是否需要额度耗尽时的通知功能（如推送通知到手机）？
