## 背景

实验室在 Azure 云上有多台虚拟机（VM），包含 GPU 实例。用户需要快速了解这些服务器的运行状态（是否正在运行、是否已被 deallocated 等），以及 GPU 资源的使用情况（GPU 利用率、显存占用）。此外，需要提供一个注册机制，让用户可以将新的 Azure 服务器添加到监控面板中。

## 用户故事

- 作为实验室成员，我希望在面板上一眼看到所有 Azure 服务器的运行状态，以便决定是否需要启动/停止某台机器。
- 作为实验室成员，我希望看到 GPU 服务器的 GPU 利用率和显存占用，以便判断是否有空闲 GPU 可以使用。
- 作为实验室成员，我希望通过 API 注册新的 Azure 服务器的 SSH 地址，以便将新机器加入监控。

## 功能需求

### Azure VM 状态监控

- [ ] 展示每台 Azure VM 的基本信息（名称、区域、VM 大小等）
- [ ] 展示每台 VM 的运行状态（Running / Stopped / Deallocated / 其他）
- [ ] 使用颜色或图标直观区分不同状态
- [ ] 支持展示多台 VM 的列表视图

### GPU 监控

- [ ] 展示 GPU 服务器的 GPU 利用率（百分比）
- [ ] 展示 GPU 服务器的显存占用（已用/总量）
- [ ] GPU 数据应定时刷新

### 服务器注册 API

- [ ] 提供 API 端点，允许用户注册新的 Azure 服务器
- [ ] 注册时需提供 SSH 地址信息
- [ ] 已注册的服务器信息需持久化存储
- [ ] 提供 API 端点查询已注册的服务器列表
- [ ] 提供 API 端点删除已注册的服务器

## 非功能需求

- **性能**: GPU 数据采集间隔应合理（避免过于频繁给服务器造成负担，也不能太慢失去参考价值）
- **兼容性**: 遵循 REQ-001 的整体约束
- **安全性**: SSH 凭证和 Azure 凭证需安全存储，不能明文暴露在 API 响应中
- **可靠性**: 当某台服务器不可达时，应优雅处理并展示"不可达"状态，而非让整个面板崩溃

## 验收标准

- [ ] 面板正确展示至少一台 Azure VM 的运行状态
- [ ] VM 状态变化（如从 Running 变为 Deallocated）能在合理时间内反映到面板上
- [ ] 面板正确展示至少一台 GPU 服务器的 GPU 利用率和显存占用
- [ ] 通过注册 API 成功添加一台新服务器后，该服务器出现在监控面板上
- [ ] 通过 API 删除一台服务器后，该服务器从监控面板消失
- [ ] 当某台服务器不可达时，面板显示相应的错误/离线状态

## 开放问题

- Azure VM 状态的获取方式：通过 Azure API（需要 Azure 凭证）还是通过 SSH 连接检测？
  - 如果用 Azure API：需要哪些权限？Service Principal 还是 Managed Identity？
  - 如果用 SSH：如何判断 deallocated 状态（SSH 连不上可能是网络问题也可能是关机）？
- GPU 数据采集方式：通过 SSH 远程执行 `nvidia-smi`？还是在目标机器上部署一个轻量 agent 上报数据？
- "注册 SSH 地址" 具体包含哪些字段？仅 host:port？还是包含用户名、密钥路径等？
- 注册 API 是否需要认证？谁都可以注册还是需要某种权限？
- 是否需要展示 CPU/内存/磁盘等基础资源信息，还是只关注 VM 状态和 GPU？
- 是否需要展示每张 GPU 卡的独立信息（多卡服务器场景）？
- 是否需要历史趋势数据（如过去 24h 的 GPU 利用率曲线），还是只看当前实时值？
- Azure VM 的数量大约是多少台？（影响轮询策略和前端展示设计）
