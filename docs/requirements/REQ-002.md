## 背景

实验室在 Azure 云上有多台虚拟机（VM），包含 GPU 实例。用户需要快速了解这些服务器的运行状态（是否正在运行、是否已被 deallocated 等），以及 GPU 资源的使用情况（GPU 利用率、显存占用）。此外，需要提供一个注册机制，让用户可以将新的 Azure 服务器添加到监控面板中。

**已确认的技术方向**：Azure VM 状态通过 Azure API（azure-cli）获取，权限为只读（只查看不操作）。

## 用户故事

- 作为唯一用户，我希望在面板上一眼看到所有 Azure 服务器的运行状态，以便决定是否需要启动/停止某台机器。
- 作为唯一用户，我希望看到 GPU 服务器的 GPU 利用率和显存占用，以便判断是否有空闲 GPU 可以使用。
- 作为唯一用户，我希望通过 API 注册新的 Azure 服务器的 SSH 地址，以便将新机器加入监控。
- 作为唯一用户，我希望看到服务器资源的历史趋势（如 GPU 利用率随时间的变化），以便了解使用规律。

## 功能需求

### Azure VM 状态监控

- [ ] 通过 Azure API（azure-cli）获取 VM 状态，权限为只读（不执行任何操作性 API 调用）
- [ ] 展示每台 Azure VM 的基本信息（名称、区域、VM 大小等）
- [ ] 展示每台 VM 的运行状态（Running / Stopped / Deallocated / 其他）
- [ ] 使用颜色或图标直观区分不同状态
- [ ] 支持展示多台 VM 的列表视图

### GPU 监控

- [ ] 展示 GPU 服务器的 GPU 利用率（百分比）
- [ ] 展示 GPU 服务器的显存占用（已用/总量）
- [ ] GPU 数据应定时刷新

### 历史数据

- [ ] 持久化存储 GPU 利用率、显存占用等监控数据的历史记录
- [ ] 支持历史趋势展示（如 GPU 利用率随时间变化的趋势图）

### 服务器注册 API

- [ ] 提供 API 端点，允许用户注册新的 Azure 服务器
- [ ] 注册时需提供 SSH 地址信息
- [ ] 已注册的服务器信息需持久化存储
- [ ] 提供 API 端点查询已注册的服务器列表
- [ ] 提供 API 端点删除已注册的服务器

## 非功能需求

- **性能**: GPU 数据采集间隔应合理（避免过于频繁给服务器造成负担，也不能太慢失去参考价值）
- **兼容性**: 遵循 REQ-001 的整体约束
- **安全性**: SSH 凭证和 Azure 凭证需安全存储，不能明文暴露在 API 响应中；Azure API 权限限制为只读
- **认证**: 无需 API 认证（单用户系统，通过 Tailscale 内网隔离保护）
- **可靠性**: 当某台服务器不可达时，应优雅处理并展示"不可达"状态，而非让整个面板崩溃
- **数据持久化**: 需要持久化存储历史监控数据以支持趋势图功能

## 验收标准

- [ ] 面板正确展示至少一台 Azure VM 的运行状态
- [ ] VM 状态变化（如从 Running 变为 Deallocated）能在合理时间内反映到面板上
- [ ] 面板正确展示至少一台 GPU 服务器的 GPU 利用率和显存占用
- [ ] 通过注册 API 成功添加一台新服务器后，该服务器出现在监控面板上
- [ ] 通过 API 删除一台服务器后，该服务器从监控面板消失
- [ ] 当某台服务器不可达时，面板显示相应的错误/离线状态
