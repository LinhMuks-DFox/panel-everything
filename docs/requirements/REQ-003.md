## 背景

实验室使用 Tailscale 组建虚拟专网，多台计算服务器通过 Tailscale 互联。用户需要在面板上查看这些 Tailscale 节点的在线状态，以便了解哪些计算服务器当前可用。

Tailscale 同时也是整个系统的访问控制基础 — Panel Everything 仅在 Tailscale 网络内可访问，不需要额外的认证机制（参见 REQ-001 访问控制章节）。

## 用户故事

- 作为唯一用户，我希望在面板上看到所有 Tailscale 节点的在线/离线状态，以便快速判断哪些计算服务器当前可以连接。
- 作为唯一用户，我希望看到每个节点的基本网络信息，以便方便地发起连接。

## 功能需求

- [ ] 展示 Tailscale 网络中所有节点的列表
- [ ] 展示每个节点的在线/离线状态
- [ ] 展示每个节点的基本信息（主机名、Tailscale IP、操作系统等）
- [ ] 使用颜色或图标直观区分在线/离线状态
- [ ] 节点状态应定时刷新

## 非功能需求

- **性能**: 遵循 REQ-001 的轻量级约束
- **兼容性**: 遵循 REQ-001 的多终端适配要求
- **安全性**: Tailscale API Key 需安全存储，不能暴露在前端
- **认证**: 无需 API 认证（单用户系统，Tailscale 内网隔离）

## 验收标准

- [ ] 面板正确展示 Tailscale 网络中的节点列表
- [ ] 在线节点和离线节点可被正确区分
- [ ] 节点上线/下线后，面板状态在合理时间内更新
- [ ] 展示的 Tailscale IP 与实际一致

## 依赖

- REQ-001（系统整体约束）

## 开放问题

- Tailscale 数据获取方式：使用 Tailscale API（需要 API Key + Tailnet 名称）还是在树莓派本地执行 `tailscale status` 命令？
  - 如果用 API：树莓派是否在同一 Tailscale 网络中？
  - 如果用本地命令：树莓派是否已加入 Tailscale 网络？
- "计算服务器" 是指 Tailscale 网络中的所有节点，还是需要筛选/标记特定的节点？
- 是否需要展示节点的最后在线时间？
- 是否需要展示节点间的连接质量/延迟？
- 是否需要区分节点类型（如服务器 vs. 个人设备）？是否需要过滤或分组功能？
- Tailscale 节点与 REQ-002 中的 Azure 服务器是否有重叠？如果有，是否需要在面板上关联展示？
