## 目标

在 MS-001 底座上叠加两个独立监控模块，实现项目的核心价值交付：在单屏总览上同时呈现 Azure VM 电源状态、GPU 利用率/显存占用、Tailscale 节点在线态。单台设备不可达时面板优雅降级，不影响其他模块；所有数据持久化到 SQLite。

本里程碑完成后，用户可在 Kindle/iPhone/iPad 上一眼看到实验室所有设备的运行状态，实现"打开即看，零操作"的核心体验。

## 交付检查

### Azure VM + GPU（TASK-010~015）

- [ ] `POST /api/v1/servers` 成功注册一台服务器，响应 201，`ssh_key_path` 字段不出现在响应中
- [ ] `GET /api/v1/servers` 返回已注册服务器列表
- [ ] `DELETE /api/v1/servers/{id}` 成功删除，返回 204
- [ ] Azure VM 采集器按配置间隔触发，`collector_run` 记录状态为 `up`
- [ ] `GET /api/v1/dashboard/azure` 返回至少一台 VM 的电源状态（`running`/`deallocated`/`stopped` 等）
- [ ] VM 状态变化在下一个采集周期内反映到 API 响应
- [ ] GPU 采集器通过 SSH 连接目标服务器，`nvidia-smi` 多卡数据正确解析落库
- [ ] GPU 不可达时（SSH 超时/拒绝）该机状态为 `unreachable`，其他机器不受影响
- [ ] Deallocated VM 对应的 GPU 采集跳过（不发起 SSH，不产生超时错误）
- [ ] VmCard 在前端正确展示 VM 名称、状态（色+形符+文字三重编码）
- [ ] GpuCard 展示 GPU 利用率进度条、显存占用（已用/总量）
- [ ] e-ink 模式下无 box-shadow、无动画

### Tailscale 网络监控（TASK-020~022）
